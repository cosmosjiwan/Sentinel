# Sentinel DLP — AI 문서 보안 검열

LLM 기반 내부 문서 민감정보 자동 탐지 및 비식별화 시스템

## 주요 기능

- **AI 민감정보 탐지 + ISO 27005 위험성 평가** — 등급별 검열(특급=차단 / 1급=마스킹 / 2급=치환 / 3급=공개)
- **검토자 검토 화면** — 탐지 항목별 검열 방식 전환·치환 표현 직접 수정, 탐지 근거 및 기밀성/무결성/가용성(C·I·A) 분해 표시
- **검토 도구** — 등급 필터·정렬, 표시 항목 일괄 적용(마스킹/치환/공개)
- **수동 추가 검열** — 본문에서 텍스트를 드래그해 AI가 놓친 항목을 직접 추가/삭제
- **검열본 내보내기** — TXT / PDF / DOCX (한글 포함, 단락 구조 유지)
- **검열 이력 & 대시보드** — 과거 결과 재방문, 위험 현황·등급 분포 통계
- **정책 설정 UI** — 점수 가중치·등급 임계값·커스텀 정규식·키워드 사전을 코드 수정 없이 조정 (`policy.json`)

## 작동 방식

### 전체 흐름

```
업로드(.txt/.md/.pdf/.docx)
   │
   ▼  ① 정규식 1차 탐지 (patterns.py)      ── 이메일·전화·주민번호·API키·카드/계좌 등
   ▼  ② LLM 검열 + 위험성 평가 (gpt-5.1)   ── 1차 탐지 결과 + 정책을 힌트로 주입
   ▼  ③ 위험성 평가 → 등급 산출 (risk.py)  ── ISO 27005 C·I·A 가중합 점수 → 등급
   ▼  ④ 등급별 처리 (app.py)               ── 특급=차단 / 1급=마스킹 / 2급=치환 / 3급=공개
   ▼  ⑤ 결과 저장 (results/<doc_id>.*)     ── JSON + TXT + PDF + DOCX
   │
   ▼  ⑥ 검토 화면(output.html)             ── 담당자가 항목별 검열 방식·표현 수정, 누락 항목 추가
   ▼  ⑦ 검열본 다운로드                     ── 검토 반영 후 TXT/PDF/DOCX 내보내기
```

업로드 후 검열은 서버에서 **gpt-5.1 호출 1회**로 한 번에 처리됩니다. 텍스트 파일(.txt/.md)은
본문을 직접 읽어 정규식 사전탐지에 활용하고, .pdf/.docx는 OpenAI Files API로 업로드해 처리합니다.

### 1) 민감정보 탐지 — 정규식 + LLM 2단계

- **정규식 1차 탐지(`patterns.py`)**: 이메일·전화번호·주민번호·OpenAI/AWS/GitHub 키·비밀번호·카드/계좌번호·IP를
  결정론적으로 찾아냅니다. 정책에 등록된 **커스텀 정규식**과 **키워드 사전**도 함께 적용됩니다.
- **LLM 2차 탐지(gpt-5.1)**: 1차 탐지 결과를 힌트로 받아 문맥 기반으로 인물·기관·지적재산·인증·금융
  정보를 탐지하고, 본문의 각 민감정보를 `<R번호>치환표현</R번호>` 태그로 감쌉니다. 탐지 항목별로
  유형·등급·점수·원문·치환표현·**판단 근거**를 표로 함께 산출합니다.
- **잔여 마스킹 안전망**: LLM이 태그로 감싸지 못하고 평문으로 남은 정규식 매칭값은 서버에서 강제 마스킹합니다.

### 2) 위험성 평가 (ISO 27005)

문서 전체를 **기밀성(C)·무결성(I)·가용성(A)** 3축으로 각 0~100점 평가한 뒤, 가중 합산하여
0~100점의 **민감성 점수**를 산출하고 점수 구간에 따라 등급을 매깁니다. 가중치와 임계값은 정책에서 조정 가능합니다.

```
민감성 점수 = C×0.6 + I×0.2 + A×0.2   (기본 가중치, DLP 특성상 기밀성 비중 ↑)
```

| 등급 | 기본 임계값 | 처리 방식 | 의미 |
|------|------------|-----------|------|
| 특급 | 90점 이상 | **차단(block)** | 문서 내용·다운로드 자체를 제공하지 않음 |
| 1급 | 70점 이상 | **마스킹(mask)** | `[비공개]`로 완전 가림 |
| 2급 | 40점 이상 | **치환(substitute)** | 의미가 유지되는 표현으로 대체 |
| 3급 | 40점 미만 | **공개(publish)** | 원문 그대로 노출 |

> 등급별 점수·처리 방식은 **항목 단위**로도 적용되며, 검토자가 항목마다 방식을 바꾸면 등급이 함께 재분류됩니다.

### 3) 마커(태그) 메커니즘

검열된 본문은 `<R번호>치환표현</R번호>` 형식의 태그로 민감정보를 표시합니다. 동일 원문은 문서 전체에서
같은 번호를 공유합니다. 마스킹/치환 토큰은 **어떤 유형의 정보였는지 드러나지 않도록 `[비공개]`로 통일**되어
있어, 마커만 보고 개인정보·인증정보 여부를 역추정할 수 없습니다.

### 4) 검토 화면 (Human-in-the-loop)

- **좌측 문서 / 우측 검토 패널**의 2-컬럼 구성. 본문의 검열 토큰과 우측 카드가 클릭 시 서로 강조됩니다.
- **검열 방식 전환**(마스킹/치환/공개): 방식을 바꾸면 등급이 재분류되고 보고 있던 등급 탭도 새 등급으로 따라 이동합니다.
- **치환 표현·판단 근거 직접 수정**, 등급 **필터·정렬**, 표시 항목 **일괄 적용**을 지원합니다.
- **수동 추가 검열**: 본문에서 텍스트를 드래그해 AI가 놓친 항목을 검열 방식·판단 근거와 함께 추가하거나 삭제할 수 있습니다.
- **원문 대조 토글**로 검열 전/후를 비교합니다.

### 5) 결과 저장 & 내보내기

검열 결과는 고유 `doc_id` 기준으로 `results/` 에 저장됩니다(동일 파일명 재업로드 시 이력이 덮어쓰이지 않음).

| 파일 | 내용 |
|------|------|
| `<doc_id>.json` | 본문·탐지 항목·위험성 평가·메타데이터 (검토/이력의 원본) |
| `<doc_id>_redacted.txt` | 평문 검열본 |
| `<doc_id>_redacted.pdf` | 한글 PDF 검열본 (단락 구조 유지) |
| `<doc_id>_redacted.docx` | Word 검열본 |

검토자가 다운로드하면 현재 검토 상태(본문 + 항목)가 서버에 반영된 뒤 해당 포맷이 재생성되어 내려받아집니다.

### 6) 정책 설정 (`policy.json`)

`정책 설정` 화면에서 코드 수정 없이 아래를 조정하며, 저장 시 **이후 검열부터** 즉시 반영됩니다.

- 위험성 점수 **가중치**(C/I/A, 저장 시 합이 1이 되도록 정규화)
- 등급 **임계값**(특급/1급/2급)
- **커스텀 탐지 정규식** 및 **민감 키워드 사전** (현재 적용 중인 기본 규칙은 예시와 함께 읽기 전용 표시)

### 라우트 요약

| 메서드 · 경로 | 설명 |
|---------------|------|
| `GET /` | 업로드 화면 |
| `POST /redact` | 파일 검열 → 결과 목록 |
| `GET /view/<doc_id>` | 검열 결과 검토 화면 |
| `POST /apply_review/<doc_id>` | 검토 상태(본문+항목) 저장 후 검열본 재생성 |
| `GET /download/<doc_id>/<txt\|pdf\|docx>` | 검열본 다운로드 |
| `GET /dashboard` | 검열 이력 & 위험 현황 대시보드 |
| `GET /policy`, `POST /policy` | 정책 조회 / 저장 |

## 프로젝트 구조

```
sentinel-dlp/
├── app.py                 # Flask 백엔드 (라우팅·검열 파이프라인)
├── patterns.py            # 정규식 1차 탐지 + 정책 커스텀 규칙/사전
├── risk.py                # ISO 27005 점수·등급 산출 (정책 기반)
├── policy.py              # 정책(policy.json) 로드/저장
├── export.py              # 검열본 PDF/DOCX 생성
├── templates/
│   ├── input.html         # 파일 업로드 페이지
│   ├── loading.html       # 검열 진행 오버레이
│   ├── list.html          # 일괄 검열 결과 목록
│   ├── output.html        # 검열 결과 검토 화면
│   ├── dashboard.html     # 검열 이력 & 대시보드
│   └── policy.html        # 정책 설정
├── static/css/            # 화면별 스타일
├── requirements.txt       # Python 패키지
├── render.yaml            # Render 배포 설정
└── .gitignore
```

## 배포 방법 (Render — 무료)

### 1단계: GitHub 레포지토리 생성

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/sentinel-dlp.git
git branch -M main
git push -u origin main
```

### 2단계: Render 연결

1. [render.com](https://render.com) 가입 (GitHub 계정으로 로그인)
2. Dashboard → **New** → **Web Service**
3. GitHub 레포지토리 `sentinel-dlp` 선택
4. 설정 자동 감지됨 (render.yaml 기반). 아래만 확인:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app --bind 0.0.0.0:$PORT`
5. **Environment Variables** 탭에서 추가:
   - Key: `OPENAI_API_KEY`
   - Value: `sk-proj-...` (본인의 OpenAI API 키)
6. **Deploy** 클릭

배포 완료 후 `https://sentinel-dlp.onrender.com` 형태의 URL이 생성됨.

### 로컬 실행 (개발용)

```bash
pip install -r requirements.txt
export OPENAI_API_KEY="sk-proj-..."   # macOS/Linux
# set OPENAI_API_KEY=sk-proj-...     # Windows CMD
python app.py
```

http://localhost:5000 에서 접속.

## 주의사항

- **API 키는 절대 코드에 하드코딩하지 마세요.** 환경변수(`OPENAI_API_KEY`)로 관리합니다.
- Render 무료 티어는 15분 비활성 시 서버가 슬립됩니다. 첫 요청 시 ~30초 대기가 있을 수 있습니다.
- `uploads/`와 `results/` 디렉토리는 서버 시작 시 자동 생성됩니다.
